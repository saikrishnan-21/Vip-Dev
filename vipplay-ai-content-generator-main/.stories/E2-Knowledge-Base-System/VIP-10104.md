# VIP-10104: Add Google Trends Source

**Story ID**: VIP-10104
**Story Type**: Story
**Epic**: E2-Knowledge-Base-System
**Priority**: High
**Story Points**: 5
**Status**: To Do

## User Story

As a user, I want to track Google Trends so that I can write about trending topics

## Acceptance Criteria

- [ ] **AC1**: User can enter trend source name and description
- [ ] **AC2**: User can optionally select region (ISO country code) and category
- [ ] **AC3**: Trend source is saved to MongoDB with validation
- [ ] **AC4**: Duplicate trend names are prevented (same name for same user)
- [ ] **AC5**: Trend source appears in Knowledge Base UI with active status

## Technical Details

**Frontend Components:**
- Create `AddTrendsDialog` component with:
  - Trend Name (required, 2-100 chars)
  - Description (optional, textarea)
  - Region selection (optional, ISO 2-char country codes)
  - Category selection (optional, dropdown)
  - JWT authentication integration
  - Success/error toast notifications

**Backend API:**
- Create `POST /api/sources/trends` endpoint
- JWT authentication middleware
- MongoDB integration with duplicate name prevention
- Validation using `createTrendsSourceSchema`

**Data Model:**
```typescript
{
  userId: ObjectId,
  type: 'trends',
  name: string,
  description?: string,
  status: 'active',
  trendRegion?: string, // ISO 2-char code (e.g., 'US', 'GB', 'AU')
  trendCategory?: string,
  articlesCount: 0,
  createdAt: Date,
  updatedAt: Date
}
```

**Validation:**
- Trend name: 2-100 characters, required
- Description: optional
- Region: 2-character ISO code, optional
- Category: string, optional
- Prevent duplicate trend names per user
